## Changes Made

### 1. Fixed: Test 6 — Content Handling (`6_content_handling_test.js`)

**Problem:** "Should call subscriber and record" triggers a real phone call requiring human interaction. "Should configure outbound call with CTR" sends a real outbound campaign.

**Fix:** Marked both tests as `it.skip()` with documentation explaining they require manual execution:
- `it.skip('Should call subscriber and record (MANUAL — requires physical phone)')`
- `it.skip('Should configure outbound call with CTR (MANUAL — outbound campaign)')`

**Also fixed:** Inverted assertion in `content_handling_objects.js` — changed `should('not.have.text', ' Call Successful... ')` to `should('have.text', ' Call Successful... ')` with a 30s timeout, so the assertion is correct if someone un-skips the test.

### 2. Fixed: Regression Test — Flow Block (`flow_block_test.js`)

**Problem 1:** All 5 tests failed because `ThankYouHaveANiceDay.wav` doesn't exist in the audio library on darkmatter.

**Fix:** Updated all 3 fixture files to use `"Recorded by Cypress"` (which exists in the org's audio library):
- `flow_message_block_details.json`
- `flow_numeric_block_details.json`
- `flow_openendedq_block_details.json`

**Problem 2:** Tests 4 ("Should edit created Flow") and 5 ("Clean up test") failed because the flow builder SPA doesn't have the main navigation sidebar. After publishing in test 3, the session context was lost for subsequent tests.

**Fix:** Added `cy.loginToVoto()` before `cy.EditFlow_tree()` in test 4 and before `cy.DeleteFlow()` in test 5 to re-establish the session and navigate back to the main app.

**Problem 3:** "Should edit created Flow" timed out waiting for `[data-cy="builder-toolbar--flow-details--btn"]` (default 10s timeout insufficient for flow builder page load).

**Fix:** Increased timeout to 20s with visibility check in `flow_block_objects.js`:
```javascript
cy.get('[data-cy="builder-toolbar--flow-details--btn"]', { timeout: 20000 }).should('be.visible').click()
```

## Recommendations for Future Work

1. **Test 6 — Call to Record:** Consider using `cy.intercept()` to mock the telephony API for automated runs, or set up a SIP loopback endpoint for automated call testing
2. **Test 6 — Outbound Campaign:** Could be automated if a test sandbox is available that doesn't consume telephony credits
3. **Audio library dependency:** The regression test depends on "Recorded by Cypress" existing in the audio library. Consider adding a setup step that uploads a test audio file if it doesn't exist
4. **Flow builder navigation:** The flow builder is a separate SPA — any test that transitions between main app and flow builder should re-login or use `cy.visit()` to reset navigation state
